프로젝트 개요

문제 정의

비효율적인 수작업 프로세스

온라인 교육 사업 확장에 따라 튜터 채용이 증가하고 있으나, 계약서 요청부터 온보딩까지의 전 과정이 수작업으로 이루어져 처리 시간이 과도하게 소요되고 있습니다.

분산된 튜터 정보 관리

튜터의 경력 정보, 전문 분야, 기술 스택 등 중요한 정보가 여러 시스템에 분산되어 있어 효율적인 관리와 활용이 어려운 상황입니다.

계약 진행 상황 추적 한계

외부 전자계약 SaaS 사용으로 인한 API 연동 제약과 수동 상태 관리로 인해 실시간 진행 상황 파악이 어려워 업무 효율성이 저하되고 있습니다.

확장성 부족

현재 시스템으로는 향후 튜터 규모 확장 시 대응이 어려우며, 튜터 평가, 성과 관리 등 추가 기능 개발에 한계가 있습니다.

해결 방안

내부 운영 통합 관리 시스템 구축

내부 운영진을 위한 전용 시스템을 구축하여 계약서 요청부터 온보딩까지의 전 과정을 디지털화하고, 튜터 정보를 중앙 집중식으로 관리하여 운영 효율성을 극대화합니다.

API 명세

POST /api/v1/contract-requests
새로운 계약서 요청 생성
Request Body:
  • name (string, required) - 튜터 이름
  • email (string, required) - 이메일
  • phone (string, required) - 연락처
  • tutor_type (enum, required) - 튜터 유형
GET /api/v1/contract-requests
계약서 요청 목록 조회
Query Parameters:
  • page (int) - 페이지 번호
  • limit (int) - 페이지 크기
  • status (enum) - 상태 필터
PUT /api/v1/contracts/{id}/status
계약서 상태 업데이트
Request Body:
  • status (enum, required) - 새로운 상태
  • notes (string) - 상태 변경 메모
GET /api/v1/tutors/{id}/profile
튜터 프로필 조회
Response:
  • 기본 정보 (이름, 연락처, 이메일)
  • 경력 정보 (년차, 기술스택)
  • 계약 상태 및 진행 상황

데이터베이스 스키마

tutors (튜터 정보)

  • tutor_id INT PK
  • name VARCHAR(100) NOT NULL
  • email VARCHAR(200) UNIQUE
  • phone VARCHAR(20)
  • tutor_type ENUM NOT NULL
  • status ENUM NOT NULL
  • created_at TIMESTAMP

contract_requests (계약서 요청)

  • request_id INT PK
  • tutor_id INT FK
  • manager_id INT NOT NULL
  • contract_type ENUM NOT NULL
  • status ENUM NOT NULL
  • requested_at TIMESTAMP

external_contracts (외부 계약서)

  • contract_id INT PK
  • request_id INT FK
  • external_platform VARCHAR(50)
  • status ENUM NOT NULL
  • sent_at TIMESTAMP
  • signed_at TIMESTAMP

managers (매니저)

  • manager_id INT PK
  • name VARCHAR(100) NOT NULL
  • email VARCHAR(200) UNIQUE
  • role ENUM NOT NULL
  • is_active BOOLEAN DEFAULT TRUE
